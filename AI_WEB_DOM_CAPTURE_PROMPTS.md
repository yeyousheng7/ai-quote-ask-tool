# AI 网页 DOM 采集提示词

用途：在目标 AI 网页版中依次发送下面的提示词，等待每一轮完整生成后，将最终长对话页面保存为对应示例文件，例如 `deepseek-示例.html`、`claude-示例.html` 或其它 provider 名称，用于后续适配插件 DOM driver。

建议：

- 使用一个全新的会话。
- 每条提示词单独发送，不要合并。
- 如果目标网页提供“深度思考 / 推理 / Thinking / Reasoning”等模式，请至少用思考模式发送最后两条。
- 保存网页前，保留默认折叠状态；如果有“展开思考 / 查看推理”按钮，可以另外抓取该按钮 DOM。
- 不需要追求答案正确性，重点是让页面出现各种结构。
- 控制回复长度。除“长回复与流式输出”外，每次回答尽量不超过 300 字，避免保存的 HTML 过大。

## 1. 普通短回复

```text
请用一句中文回复：这里一切正常。
```

## 2. Markdown 层级与列表

```text
请用 Markdown 写一段关于“浏览器插件批注提问”的说明，必须包含：

1. 一个二级标题
2. 一个三级标题
3. 一个无序列表，至少 3 项
4. 一个有序列表，至少 3 项
5. 加粗、斜体、删除线
6. 一句总结

请控制在 250 字以内。
```

## 3. 行内代码与普通文本混排

```text
请解释这句话中的几个术语，并保留行内代码格式：

`mypipe 数组长度太短`、`PIPE_BUF`、`select()`、`FD_SET`、`errno`

要求：
- 每个术语单独解释
- 解释中继续穿插一些行内代码
- 不要使用代码块
- 总字数不超过 250 字
```

## 4. 多语言代码块

```text
请给出一个 Linux C 程序片段，演示 `select()` 同时监听两个文件描述符。

要求：
- 使用 fenced code block
- 代码块语言标记为 c
- 代码中包含 `FD_ZERO`、`FD_SET`、`select`
- 代码后再给一个 bash 命令代码块
- 最后用两段中文解释
- C 代码不超过 35 行，解释不超过 180 字
```

## 5. 表格

```text
请用 Markdown 表格比较“管道、FIFO、消息队列、共享内存、信号量”。

表格列包括：
- IPC 方式
- 是否有名
- 适合场景
- 常见问题

表格后再补充 3 条注意事项。

请让每个单元格内容尽量简短。
```

## 6. 数学公式

```text
请解释主定理，并渲染以下公式：

行内公式：T(n) = 2T(n/2) + n

块级公式：
T(n) = aT(n/b) + f(n)

再给出一个分段函数和一个求和公式。请尽量使用网页会渲染的数学公式格式。

解释文字控制在 180 字以内。
```

## 7. 引用块与分割线

```text
请写一段学习笔记，主题是“为什么死锁会发生”。

要求：
- 开头有一句普通正文
- 中间有一个引用块
- 引用块后有一条分割线
- 分割线后继续写 2 段正文
- 总字数不超过 250 字
```

## 8. 复杂混合回复

```text
请生成一份“小型网络服务端的安全检查清单”。

必须包含：
- 标题
- 简短导语
- 无序列表
- 有序列表
- 表格
- 一段 C 代码块
- 一段 Bash 代码块
- 行内代码
- 引用块
- 最后一行总结

请保持紧凑：代码块不超过 25 行，表格不超过 4 行，总字数不超过 500 字。
```

## 9. 长回复与流式输出

```text
请写一篇中等长度的中文说明，主题是“为什么浏览器插件在适配不同 AI 网页时需要 provider driver 设计”。

要求：
- 约 600 字
- 分成 4 个小节
- 每个小节有标题
- 中间穿插列表、行内代码和表格
- 结尾给出 3 条工程建议
```

## 10. 思考模型测试

如果目标网页提供思考模式，请开启思考模式后发送：

```text
请先推理再回答：一个浏览器插件需要在 AI 网页中支持“划词提问、右侧小窗显示、主对话隐藏临时消息、刷新后恢复标记”。请分析可能的 DOM 适配难点，并给出实现方案。

回答要求：
- 展示清晰的最终答案
- 包含列表和小标题
- 包含至少一个代码块
- 包含一个表格
- 最终答案控制在 600 字以内，代码块不超过 20 行
```

## 11. 多轮追问上下文

继续在同一个会话里发送：

```text
基于你上一条回答，只讨论“主对话隐藏临时消息”这个点。请说明：

1. 什么时候应该隐藏
2. 什么时候应该卸载 DOM
3. 如果页面刷新，如何恢复
4. 如果 DOM 结构变化，如何降级

请控制在 400 字以内。
```

然后再发送：

```text
继续上一个问题。请把方案压缩成 5 条最重要的规则，每条不超过 30 字。
```
